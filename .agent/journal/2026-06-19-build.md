# Build Journal: Neon Pipes VR (#57)
**Date:** 2026-06-19 (AM cycle)
**Duration:** ~45 minutes, 1 round

## Round 1 — Full Build + Deploy

### Concept
Pipe rotation/flow puzzle — Pipe Dream / Pipe Mania style. Rotate pipe segments on a grid to connect source to drain. New genre (#46 in the portfolio).

### Implementation
- Scaffolded from neon-connect (clean IWSDK 0.4.x base) since `npx @iwsdk/create` still fails with fetch error
- 11 pipe types: straight-H, straight-V, 4 elbows, 4 T-pipes, 1 cross
- Each type has a set of connection ports (N/S/E/W) and rotation maps (CW/CCW)
- Grid generation: random walk from source to drain, then fill empty cells with random pipes in random rotations
- BFS flow validation: starts at source, follows connected ports through pipes
- 8 game modes, 3 difficulty levels (5×5, 7×7, 9×9)
- Combo system with decay timer — fast rotations build multiplier
- 40 achievements, 8 pipe skins (unlock via milestones), 5 color themes
- XP/level system with 50 levels
- Daily challenges with seeded PRNG
- 15 PanelUI panels, zero HTML DOM
- Procedural audio: rotation clicks, connection SFX, ambient drone
- Holodeck environment with grid floor/ceiling, floating decorations

### Fixes Applied
1. Removed `require()` call (CJS not valid in ESM)
2. Fixed `unProject` → `unproject` (Three.js case sensitivity)
3. Added `as any` casts for `file:` link type resolution issues (world.input.xr.gamepads, playerSpaceEntities, World.create options)
4. Fixed `FollowerMode` → `FollowBehavior` (correct IWSDK API)
5. Removed unused imports (Quaternion, ScreenSpace)
6. Fixed ambient particle opacity assignment syntax
